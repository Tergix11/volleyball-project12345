# Деплой VolleyballProject на Railway

## Что нужно сделать перед загрузкой

В архиве есть несколько проблем, из-за которых проект не запустится на Railway "как есть":
лежит локальная виртуальная среда `myenv` (которую вообще не нужно загружать),
в `settings.py` зашиты пароль от MySQL и `DEBUG = True`, нет `requirements.txt`
и файла, который говорит Railway, как запускать сервер. Ниже — готовые файлы,
которые это исправляют, плюс шаги, что делать дальше.

## 1. Файлы из этого пакета — куда их положить

Распакуй `VolleyballProject` и положи файлы из этого пакета так:

- `requirements.txt` → в корень проекта, рядом с `manage.py`
- `Procfile` → в корень проекта, рядом с `manage.py`
- `.gitignore` → в корень проекта, рядом с `manage.py`
- `settings.py` → заменить файл `volleyball_site/settings.py`
- `urls.py` → заменить файл `volleyball_site/urls.py`

Папку `myenv` из проекта можно просто удалить — она не нужна (это локальное
виртуальное окружение Python, Railway создаёт своё).

## 2. Залить проект на GitHub

Railway разворачивает проект из GitHub-репозитория. Если репозитория ещё нет:

1. Создай новый репозиторий на github.com (можно приватный).
2. В папке проекта выполни:
   ```
   git init
   git add .
   git commit -m "Подготовка к деплою"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_НИК/ИМЯ_РЕПО.git
   git push -u origin main
   ```

`.gitignore` из пакета проследит, чтобы `myenv`, кэш и старый `db.sqlite3`
не попали в репозиторий.

## 3. Создать проект на Railway

1. Зайди на railway.com, войди через GitHub.
2. New Project → Deploy from GitHub repo → выбери свой репозиторий.
3. Railway сам определит, что это Python/Django-проект, и начнёт собирать билд.

## 4. Добавить базу данных MySQL

В проекте используется MySQL — нужна отдельная база на Railway:

1. В своём проекте на Railway: Create → Database → Add MySQL.
2. Появится отдельный сервис MySQL со своими переменными окружения
   (`MYSQL_URL`, `MYSQLHOST`, `MYSQLUSER` и т.д.).
3. Перейди в сервис с твоим Django-приложением → вкладка Variables →
   добавь переменную `MYSQL_URL`, сославшись на базу:
   `MYSQL_URL = ${{MySQL.MYSQL_URL}}`
   (Railway сам предложит автодополнение при вводе).

Обновлённый `settings.py` сам подхватит эту переменную и подключится к
MySQL на Railway — больше ничего менять не нужно.

## 5. Переменные окружения для Django-сервиса

В том же разделе Variables добавь:

- `SECRET_KEY` — любая длинная случайная строка (можно сгенерировать на
  https://djecrety.ir/)
- `DEBUG` — оставь пустым или не добавляй вообще (по умолчанию будет `False`
  в продакшене, если переменная не задана хостингом... на всякий случай
  можно явно поставить `DEBUG=False`)

Домен `RAILWAY_PUBLIC_DOMAIN` Railway подставит сам — `settings.py` уже
настроен, чтобы добавлять его в `ALLOWED_HOSTS` и `CSRF_TRUSTED_ORIGINS`
автоматически.

## 6. Первый деплой и проверка

После того как переменные добавлены, Railway пересоберёт и запустит проект.
`Procfile` из пакета делает три вещи при каждом запуске:
применяет миграции (`migrate`), собирает статику (`collectstatic`)
и запускает сервер через `gunicorn`.

Если деплой прошёл успешно — открой публичный домен сервиса (Settings →
Networking → Generate Domain, если домен ещё не создан) и проверь сайт.

## 7. Создать суперпользователя (для входа в админку)

Локального доступа к базе на Railway через консоль нет по умолчанию, но
можно выполнить разовую команду через Railway CLI:

```
npm i -g @railway/cli
railway login
railway link        # выбери свой проект
railway run python manage.py createsuperuser
```

Эта команда подключится к тем же переменным окружения, что и продакшен,
и создаст администратора прямо в базе на Railway.

## Частые проблемы

- **Bad Request (400)** после деплоя — значит `ALLOWED_HOSTS` не включает
  домен. Проверь, что Railway выдал публичный домен и переменная
  `RAILWAY_PUBLIC_DOMAIN` есть у сервиса (обычно появляется автоматически).
- **Картинки заливов/товаров не открываются** — проверь, что `urls.py`
  заменён на файл из этого пакета (там медиафайлы отдаются всегда, а не
  только при `DEBUG=True`).
- **Ошибка при collectstatic про отсутствующую папку static** — это
  исправлено в новом `settings.py` (там убрана ссылка на несуществующую
  папку `BASE_DIR/static`).
